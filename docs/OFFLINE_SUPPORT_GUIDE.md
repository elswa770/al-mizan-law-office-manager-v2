# دليل نظام الدعم بدون اتصال (Offline Support)

## 🌐 **نظرة عامة**

نظام الدعم بدون اتصال في تطبيق الميزان يسمح للمستخدمين بالاستمرار في العمل حتى عند عدم وجود اتصال بالإنترنت. النظام يتضمن:

- **Service Worker محسن** مع استراتيجيات تخزين مؤقت متقدمة
- **IndexedDB** للتخزين المحلي للبيانات
- **مزامنة تلقائية** عند عودة الاتصال
- **واجهة مستخدم** لعرض حالة الاتصال والمزامنة

## 🚀 **المميزات الرئيسية**

### 📱 **Service Worker المحسن**
- **Cache-first** للموارد الثابتة (HTML, CSS, JS)
- **Network-first** لبيانات API مع fallback للتخزين المؤقت
- **Stale-while-revalidate** للطلبات الأخرى
- **معالجة خاصة** لطلبات Firebase

### 💾 **التخزين المحلي**
- **IndexedDB** لتخزين البيانات بشكل منظم
- **تخزين مؤقت** للقضايا، العملاء، الجلسات، والمهام
- **إجراءات معلقة** للمزامنة لاحقاً
- **سجل مزامنة** لتتبع العمليات

### 🔄 **المزامنة التلقائية**
- **اكتشاف الاتصال** تلقائياً
- **مزامنة الإجراءات المعلقة** عند العودة للإنترنت
- **إعادة المحاولة** للإجراءات الفاشلة (3 محاولات)
- **سجل الأخطاء** للمشاكل التي تحدث أثناء المزامنة

### 📊 **واجهة المستخدم**
- **مؤشر حالة الاتصال** في شريط الأدوات
- **لوحة تفاصيل** مع معلومات المزامنة
- **أزرار التحكم** للمزامنة اليدوية
- **استيراد/تصدير** البيانات الاحتياطية

## 🔧 **الاستخدام في المكونات**

### استخدام الـ Hooks
```typescript
import { useOfflineStatus, useOfflineActions } from '../hooks/useOfflineStatus';

function MyComponent() {
  const status = useOfflineStatus();
  const { isOnline, syncNow, clearCache } = useOfflineActions();

  return (
    <div>
      <p>الحالة: {status?.online ? 'متصل' : 'غير متصل'}</p>
      <p>الإجراءات المعلقة: {status?.pendingActions}</p>
      <button onClick={syncNow} disabled={!isOnline}>
        مزامنة الآن
      </button>
    </div>
  );
}
```

### استخدام OfflineManager مباشرة
```typescript
import { offlineManager } from '../services/offlineManager';

// تخزين البيانات محلياً
await offlineManager.cacheData('cases', casesData);

// الحصول على البيانات المخزنة
const cachedCases = await offlineManager.getCachedData('cases');

// إضافة إجراء معلق
await offlineManager.addPendingAction({
  type: 'create',
  entity: 'case',
  data: newCaseData
});
```

// للموارد الثابتة (HTML, CSS, JS)

// ثانياً: إذا لم توجد، جلب من الشبكة وتخزينها

### 2. **Network-First**
```javascript
// لبيانات API
// أولاً: جلب من الشبكة
// ثانياً: إذا فشلت، استخدام التخزين المؤقت
// ثالثاً: تخزين الاستجابة الناجحة
```

### 3. **Stale-While-Revalidate**
```javascript
// للطلبات الأخرى

```

## 📱 **واجهة المستخدم**

### مؤشر الحالة
- **🟢 أخضر**: متصل وجاهز للمزامنة
- **🟡 أصفر**: متصل ولكن توجد إجراءات معلقة
- **🔴 أحمر**: غير متصل

### لوحة التفاصيل
- حالة الاتصال الحالية
- وقت آخر مزامنة
- عدد الإجراءات المعلقة
- إحصائيات البيانات المخزنة
- أزرار التحكم

## 🔍 **استكشاف الأخطاء**

### مشاكل شائعة وحلولها

1. **Service Worker لا يعمل**
   ```javascript
   // تحقق من تسجيل الـ Service Worker
   navigator.serviceWorker.getRegistrations().then(registrations => {
     console.log('Registrations:', registrations);
   });
   ```

2. **IndexedDB لا يعمل**
   ```javascript
   // تحقق من دعم IndexedDB
   if (!window.indexedDB) {
     console.error('IndexedDB not supported');
   }
   ```

3. **المزامنة لا تعمل**
   ```javascript
   // تحقق من حالة الاتصال
   console.log('Online:', navigator.onLine);
   
   // فرض المزامنة اليدوية
   offlineManager.syncPendingActions();
   ```

## 🚧 **التطوير المستقبلي**

### تحسينات مقترحة
1. **Background Sync API** للمزامنة في الخلفية
2. **Web Workers** لمعالجة البيانات الثقيلة
3. **Compression** لتقليل حجم البيانات المخزنة
4. **Incremental Updates** لتحديث البيانات بشكل تدريجي
5. **Conflict Resolution** لحل تعارضات البيانات

### ميزات جديدة
1. **Offline Analytics** لتحليل استخدام النظام بدون اتصال
2. **Smart Caching** للتنبؤ بالبيانات المطلوبة
3. **Collaborative Editing** للتعاون بدون اتصال
4. **Push Notifications** للإشعارات بدون اتصال
